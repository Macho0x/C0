# Goop 1.5 + Treelog native big-bang

**Date:** 2026-07-11  
**Status:** Approved for implementation (user chose approach 3)

## Goal

Ship Goop codegen fixes so OCaml-style juxtaposition (`F a b`, `f ()`) lowers correctly for capitalized / Go-imported names, then rewrite treelog so Options, New, Scope, and Handler bodies are majority Goop. Keep `@[go]` only for construction / Go-API bridges.

## Decisions (locked)

| Topic | Choice |
|-------|--------|
| Scope of migration | Big-bang: Options + New + Scope + Handle/sanitize |
| Scope error API | Goop `Fail` / `fail`; `@[go]` may expose Go method `Error` |
| Color / Pretty | `'a option` (`None` / `Some`) in Goop Options |
| Mutex | Explicit `Lock` / `Unlock` (no `defer`) |
| Syntax philosophy | No new keywords; OCaml call/option/ref surface |

## Goop codegen patches

1. **Capitalized multi-arg apps** — `Add 2 3` parses as `App(Ctor{Add,2}, 3)`. Flatten embedded ctor args when `Add` is a user `let` or Go extern (not ADT/`None`/`Some`/`Ok`/`Error`), then emit `Add(2, 3)`.
2. **Unit elision** — `Now ()` as `Ctor{Now, unit}` must emit `time.Now()`, not `time.Now(struct{}{})`. Same strip already used on field/method apps.
3. **Regressions** — e2e tests for both.

## Treelog shape

```goop
type Options = {
  service : string;
  environment : string;
  ...
  color : bool option;
  pretty : bool option;
  writer : Writer option;
}

type Scope = { ... mutable fields ... }

let Fail (s : Scope ptr) (msg : string) (attrs : Attr go_slice) : unit = ...
let Emit (s : Scope ptr) : unit = ...
(* Handle / flat_attrs / sanitize_* as Goop lets; implements Handler stays *)
```

`@[go]` retains: `new_mutex`, writer defaults (`stderr` / `discard`), `slog.New` / `NewJSONHandler` / `NewRecord` / `AddAttrs` glue if still needed, and `func (s *Scope) Error(...)` → `Fail`.

## Non-goals

- `defer` keyword
- Auto-import of all Go package exports
- Changing OCaml `#method` object semantics
